## 3. Implicit Assumptions (test limitations)
- **No real GitHub API call**: all tests use `httpx.MockTransport`. Any behavior difference
  between the mock and real GitHub (e.g. undocumented response fields, rate-limit edge
  cases not in mock) cannot be detected by the test suite.
- **Retry-After as integer seconds**: `test_retry_after_header_honored_on_429` asserts
  `slept == [5.0]` (numeric Retry-After). GitHub's actual Retry-After behavior matches,
  but date-string Retry-After (RFC 7231 format) is not tested — `float("Wed, 21 Oct...")` 
  would raise `ValueError`, falling back to computed backoff (non-numeric path covered).
- **Dep audit incomplete**: `pip-audit`/`safety` not available; `pip check` only confirms no
  broken requirements. No CVE check possible in this environment.
- **Non-string `created_at` (BUG-1)**: the None guard is correct for all realistic inputs.
  Non-string is a theoretical dirty-data variant not seen in practice.

## 4. Risky Areas (passed but fragile)
- **`_parse_created` (client.py ~L256)**: The `value.replace("Z", "+00:00")` will raise
  `AttributeError` on any non-string `created_at`. The code around it only guards `None`.
  Low risk in production, but fragile under extreme dirty data (BUG-1).
- **TLS verify (client.py L68)**: `verify=True` is set but the test cannot confirm it
  (BUG-3). Any future refactor of `__init__` that moves this line could silently drop TLS
  verification without a test failure.
- **Shared `REPO_PATTERN`**: The `..` traversal gap is collector-local only. If a future
  component reuses `REPO_PATTERN` for URL-building purposes without the extra guard, the
  gap reappears.
