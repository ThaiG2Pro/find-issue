## 1. Implementation Patterns Used

- `JsonFileStateStore` implements the existing `osspulse.ports.StateStore` Protocol (`load() -> dict`, `save(state: dict) -> None`) with two extra concrete-adapter helpers (`is_seen`, `mark_seen`) — Protocol left untouched per AC-3-018 and ADR-001.
- Atomic write pattern: `tempfile.NamedTemporaryFile` (delete=False) in `state_path.parent` + `os.fsync` + `os.replace` — guarantees same-filesystem atomic rename (ADR-002). Crash mid-write leaves the prior valid file intact.
- Error hierarchy: `StateError` in `src/osspulse/state/errors.py`, mirrors `ConfigError` per-module pattern (ADR-003). Surfaced to CLI as `Error: <message>` on stderr with exit 1.
- Lazy-load cache: `_cache: dict | None = None` on the instance; `load()` populates it; `save()` invalidates it. Avoids redundant disk reads within a single pipeline run.
- Identity key helper `_identity_key(item_type, item_id) -> str` returns `"{item_type}:{item_id}"`. Stable, testable, decoupled from `RawItem` construction.
- `_now_utc_z() -> str` is a module-level function, patchable in tests (used by `test_remark_preserves_original_first_seen_at`).
- `Config.state_path` added as a frozen-dataclass field with default `"./.osspulse/state.json"`; existing call-sites need no update.

## 2. Deviations from Design

- **DEV-001** (minor, S4): Removed the inner `try/except Exception: raise` from `save()` — was a transparent re-raise (dead branches). Outer `except OSError` handles all I/O failures. No AC impact; ruled AC-COMPLIANT by QA.
- **OI-2** (minor, S4): `load_config` reads only `[state].state_path` from TOML (no top-level fallback). Design mentioned "(or top-level)" as an alternative. Ruled AC-COMPLIANT — AC-3-013 requires config-driven path, not a specific lookup strategy.
- No other deviations. No major/structural deviations.

## 3. Inferred Decisions

- V1 records seen state (write-only from pipeline's perspective); full delta-filtering (skipping already-seen items on collect) is deferred to V2 per assumption A-A1. The `is_seen` helper exists for that V2 use.
- `version: 1` integer in the JSON document is the forward-migration handle. V2 can detect this and run a migration path without heuristics.
- No file locking (single-operator, single-process assumption A-A3). The atomic write makes concurrent writes safe from corruption but last-writer-wins semantics apply.

## 4. Where Future Developers Should Focus

- `test_remark_preserves_original_first_seen_at`: patches `_now_utc_z` at the module level. If the function is moved to a different module, the patch target becomes stale.
- `test_save_uses_temp_then_replace`: spies on `os.replace` via monkeypatch. Would need updating if the atomic-write strategy changed.
- `test_no_network_or_cross_stage_imports`: uses `inspect.getsource()` — source-text check, would not catch `if TYPE_CHECKING:` conditional imports. Not a current issue.
- `test_unwritable_path_raises_state_error`: skipped on Windows and when running as root (correct for Linux CI/non-root).
- BUG-001 regression tests (`test_null_root_*`, `test_array_root_*`, `test_integer_root_*`, `test_string_root_*`) guard the isinstance check at `json_store.py` lines 89-93. If load() is refactored, these must remain.
