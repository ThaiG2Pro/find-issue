## 4. Self-Review Log

### Security (R-SEC-001 / R-SEC-003)
- No hardcoded secrets — `state_path` is operator config, not a secret.
- No network calls, no auth tokens touch this module.
- `StateError` messages show path and OS reason only — no file contents leaked.
- State file content: public issue identities + timestamps — no PII.

### Architecture boundary check (AC-3-017)
- `json_store.py` imports: `json`, `os`, `tempfile`, `datetime`, `pathlib` (stdlib) + `osspulse.models.RawItem` + `osspulse.state.errors.StateError`. No collector, summarizer, cache, or network imports.

### Protocol invariant (AC-3-018)
- `osspulse.ports.StateStore` is untouched — still declares exactly `load() -> dict` and `save(state: dict) -> None`.
- `is_seen` and `mark_seen` are on `JsonFileStateStore` only.

### Corrupt-vs-empty boundary (hardest edge — AC-3-009/011/012)
- `raw.strip()` check runs before `json.loads` — 0-byte and whitespace files short-circuit to empty without hitting the parser.
- `JSONDecodeError` → `StateError("state file is corrupt: ...")`.
- Bool-trap guard: `type(version) is not int` rejects `True`/`False` (which `isinstance(True, int)` would pass).

### Atomic write (ADR-002)
- `mkdir(parents=True, exist_ok=True)` runs before `mkstemp(dir=state_path.parent)` — ordering verified.
- `os.replace` is same-filesystem guaranteed (temp in parent dir).
- `os.unlink(tmp_name)` in `finally` cleans up orphan on any failure path.
