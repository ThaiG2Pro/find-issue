## 4. Bug List

### BUG-001 [HIGH] — FIXED / VERIFIED (retest 2026-06-24)

- **Classification**: [LOGIC-BUG] (×2 weight)
- **AC**: AC-3-009
- **Severity**: High
- **Original Status**: NO-GO blocker (S5, 2026-06-24)
- **Retest Status**: FIXED — verified closed by QA retest (2026-06-24)
- **Description**: `load()` did not guard `isinstance(data, dict)` after `json.loads()`. When a JSON file contained a syntactically valid but structurally non-dict root value (`null`, `[]`, `42`, `"string"`), `data.get("version")` raised `AttributeError` instead of `StateError`.
- **Fix applied** (FIX-001, src/osspulse/state/json_store.py lines 89-93): `if not isinstance(data, dict): raise StateError(f"state file is corrupt: expected a JSON object, got {type(data).__name__}")` inserted between step 3 (JSONDecodeError catch) and step 4 (version check).
- **Regression tests added**: 5 tests — `test_null_root_raises_state_error_not_attribute_error`, `test_null_root_file_not_overwritten`, `test_array_root_raises_state_error_not_attribute_error`, `test_integer_root_raises_state_error_not_attribute_error`, `test_string_root_raises_state_error_not_attribute_error` — all PASS in QA retest.
- **Verification**: QA independently ran `uv run pytest tests/ -v --tb=short --cov=src/osspulse --cov-report=term-missing --cov-branch` — 119/119 pass, json_store.py 100% lines and branches including the new isinstance branch.

No new bugs introduced by the fix.
