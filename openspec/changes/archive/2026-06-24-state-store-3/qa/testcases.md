## QA-Independent Scenario Notes

**TC-021 / TC-022 — BUG-001 FIXED (retest 2026-06-24):**

Original finding: when a state file contained syntactically valid JSON with a non-dict root value (`null`, `[]`, `42`), `json.loads()` succeeded and returned a non-dict Python object, then `data.get("version")` raised `AttributeError`.

Fix (FIX-001): `if not isinstance(data, dict): raise StateError(f"state file is corrupt: expected a JSON object, got {type(data).__name__}")` inserted at json_store.py lines 89-93, between the JSONDecodeError catch and the version check.

Five regression tests added and verified by QA independent run (119/119 pass):
- `test_null_root_raises_state_error_not_attribute_error` — PASS
- `test_null_root_file_not_overwritten` — PASS
- `test_array_root_raises_state_error_not_attribute_error` — PASS
- `test_integer_root_raises_state_error_not_attribute_error` — PASS
- `test_string_root_raises_state_error_not_attribute_error` — PASS

TC-021 and TC-022 now map directly to `test_null_root_raises_state_error_not_attribute_error` and `test_array_root_raises_state_error_not_attribute_error` respectively. The integer and string-root cases are covered by the additional 3 regression tests.
