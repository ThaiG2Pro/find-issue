## 1. Implementation Patterns Established

- **Cache-aside inside the adapter** (ADR-001): `LiteLLMSummarizer` holds the `SummaryCache` port reference injected at `__init__`. The `LLMClient` Protocol is unchanged. Pattern mirrors `GitHubCollector` (injected httpx client).
- **Best-effort cache via private wrappers** (ADR-004): `_cache_get` / `_cache_set` wrap the port in `try/except Exception`; the raw `RedisSummaryCache` adapter remains a faithful port that may raise. This is the only broad-except in the change, and it is scoped precisely to cache I/O.
- **Adapter-only batch helper** (ADR-005): `summarize_items(items) -> list[SummarizedItem]` owns catch-log-skip-continue. It is not on the `LLMClient` Protocol, keeping the Protocol frozen (AC-4-003).
- **Frozen config dataclass** (ADR-007): `SummarizerConfig` carries all tunables (model, timeout, input cap, sentence count, summary char cap). No secret field — api_key is a runtime-only private attribute, never in config or repr.
- **openai.APIError as the LLM error boundary** (DEV-001): verified at runtime that all litellm exceptions inherit from `openai.APIError`, not `litellm.exceptions.APIError`. All four guard tests raise real `litellm.exceptions.*` instances and pass.
- **S6 release artifacts**: `release.md` written with migration checklist (N/A — no DB), rollback plan (branch revert + optional Redis flush), 5-step post-deploy smoke test, and additive deploy strategy. `.env.example` (BUG-002) and `README.md` with RF-1 disclosure (BUG-003) both complete.

## 3. Inferred Decisions (not in original spec)

- **Newline separator in content_hash** (`title + "\n" + body`): prevents concatenation collisions (e.g. title `"AB"` + body `"C"` vs title `"A"` + body `"BC"`). Confirmed at SPEC LOCK Q1.
- **Hash over truncated text**: the 8000-char truncated text is both hashed and sent to the LLM. This keeps the cache key stable — a re-run after truncation produces the same key (AC-4-019). Confirmed at SPEC LOCK Q3.
- **No cache TTL in V1**: content-hash is the sole invalidation mechanism. An edited issue gets a new hash → fresh summary. TTL adds complexity with no V1 benefit. Confirmed at SPEC LOCK Q4.
- **`_SkipItem` as a private sentinel**: raised for fully-empty items inside `summarize()`, caught immediately inside `summarize_items()`. Never surfaces to the caller, never logged (AC-4-018).
- **`pipeline.py` not wired**: `summarize_items` is production-ready but not called from `pipeline.py`. The pipeline still raises `NotImplementedError`. This is the architectural seam the next change must close.
