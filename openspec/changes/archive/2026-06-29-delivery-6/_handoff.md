## 1. Key Decisions & Implementation Patterns

- **Atomic write pattern**: Copied exactly from `state/json_store.py:save` — `mkstemp(dir=target.parent)` → `fdopen(utf-8)` write+flush+fsync → `os.replace` → `finally: unlink`. `tmp_name = None` after successful replace prevents double-unlink.
- **No mkdir**: Deliberately dropped from state-store-3 template (AC-6-014). Missing parent causes `mkstemp` to raise `FileNotFoundError` (subclass of `OSError`) → caught → `DeliveryError` naming the path.
- **StdoutDelivery stream injection**: `__init__(stream=None)` defaults to `sys.stdout` — injectable seam for tests; avoids patching sys.stdout.
- **BrokenPipeError locus**: CLI top-level `except BrokenPipeError` → `os.dup2(devnull, sys.stdout.fileno())` → `Exit(0)`. StdoutDelivery does not catch it.
- **Digest import removed from ports.py**: `Delivery` no longer uses `Digest` after ADR-001; ruff F401 confirmed unused; safely removed. `RawItem`/`SummarizedItem` stay (used by other Protocols).

## 3. Implicit Assumptions

- `os.replace` on same-filesystem is atomic on Linux (POSIX `rename(2)`). The `mkstemp(dir=target.parent)` guarantee ensures same-fs.
- `typer.testing.CliRunner` isolates stdout/stderr per invocation; the BrokenPipeError handler cannot be easily unit-tested via CliRunner (would need real subprocess pipe).
- `os.chmod(dir, 0o444)` in permission test requires non-root execution; on CI running as root this test may pass for wrong reasons.
