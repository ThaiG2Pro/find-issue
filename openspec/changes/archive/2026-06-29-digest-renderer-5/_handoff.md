## 1. Implementation Patterns

- **Pure free function + thin adapter class**: `render()` is a module-level pure function;
  `MarkdownDigestRenderer` is a one-liner adapter delegating to it. This mirrors the
  summarizer adapter style and keeps unit tests trivially mockless.
- **Dict-of-dict deterministic grouping**: `dict[repo] -> dict[group_key] -> list[item]`
  using Python insertion order. Repos sorted once via `sorted(keys, key=str.lower)`. Items
  keep input order. No `set` used anywhere — eliminates the only non-determinism risk.
- **Fixed `GROUP_ORDER` constant**: `["issue", "discussion", "release"]`. Groups are emitted
  in this order; any unknown `item_type` falls into the trailing `__other__` / `### Khac`
  bucket. Adding a new known type in V2 requires only appending to this constant.
- **Independent omit-branches in `_build_item_line`**: each of the three optional segments
  (title, summary, url) is guarded by a truthiness check independently. Segments are
  collected in a `parts: list[str]` and joined with `" "`. Order is fixed: id, title,
  summary, url. Never raises on dirty / empty fields.
- **Keyword-only `lookback_days`**: enforced by `*` in the function signature.
  Call sites MUST NOT pass it positionally. The port contract mirrors this exactly.
- **Static import isolation**: only `osspulse.models` imported in `renderer.py`. Confirmed
  by `test_render_import_isolation.py` (AC-5-003 whitelist check via `ast.walk`).

## 4. Where Future Work Should Focus

- **Pipeline wiring**: the renderer is not yet called from `pipeline.py` or `cli.py`.
  The follow-on delivery/wiring change must honor the keyword-only contract:
  `render(items, *, lookback_days=config.lookback_days)`.
- **Markdown escaping (V2)**: field text is rendered as-is (A-A4, documented). If the
  tool ever gains untrusted-input paths (e.g., webhook delivery), escaping should be
  added to `_build_item_line`.
- **EC-004 stress test**: no permanent automated 10k-item regression test exists. If the
  rendering algorithm changes, add a scale regression at that time.
- **`runtime_checkable` Protocol**: `DigestRenderer` is a structural Protocol without
  `@runtime_checkable`. If `isinstance` checks are needed at the call site, add the
  decorator to `ports.py`.
