## Implementation Notes

- The em-dash `—` (U+2014) in `_build_item_line` is used as specified in BR-5-007. Tests explicitly assert the em-dash character is present (AC-5-012) and absent from item lines when summary is empty (AC-5-017). Two tests for AC-5-017 originally used a broad `assert "—" not in result` which incorrectly matched the em-dash in the repo header (`## r/x — 7 ngày qua`); fixed by inspecting only item lines via `_item_lines()` helper.
- The test for within-group order (AC-5-007) originally used `l.split()[0]` to extract the id token, but `"- #9".split()[0]` is `"-"` not `"- #9"`. Fixed to use `line.split()[1]` to extract the `#id` token.
- Both fixes were code-quality corrections to the tests (wrong assertion technique), not design or implementation issues.
