### 4.1 renderer.py (~90 lines)

Read top-to-bottom. Findings:

- **Constants** (`GROUP_ORDER`, `GROUP_LABELS`): correct, match BR-5-008 exactly. `__other__` key maps to `Khác`.
- **`_build_item_line`**: Composable `parts: list[str]` built with 3 independent truthiness gates. `raw.title` (non-empty string = truthy), `item.summary.strip()` (whitespace omit gate), `raw.url` (non-empty = truthy). Uses U+2014 em-dash as specified. `" ".join(parts)` — spacing correct for AC-5-012. Never raises (all branches are additive; no mutation of the item). No Markdown escaping per A-A4. Correct.
- **`render()` free function**: Single-pass `dict[repo][group_key].append()` grouping — no `set` anywhere (confirmed by static grep + spot-check). Empty-input branch returns early with the "No new items" doc, including trailing `"\n"` via the `""` append + join pattern. Repo emission via `sorted(grouped, key=str.lower)` — only repos sorted, items kept in append (input) order. Group emission via `GROUP_ORDER + ["__other__"]` — fixed, no sorting of items. AC-5-011 empty-group skip via `if not group_items: continue`. AC-5-014 count via `len(group_items)` (reflects duplicates — BR-5-010). Trailing `lines.append("")` + `"\n".join(lines)` produces a clean trailing newline. Correct.
- **`MarkdownDigestRenderer`**: One-liner delegation to free `render()`. No I/O methods. Structurally satisfies `DigestRenderer` Protocol. Correct.
- **Import isolation**: Only `from osspulse.models import SummarizedItem`. Confirmed by `ast` static test and manual grep.

No correctness issues found in the implementation.
