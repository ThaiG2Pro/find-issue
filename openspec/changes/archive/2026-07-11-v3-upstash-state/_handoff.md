## 2. Contentious Points

- **`save()` uses HSET vs `mark_seen()` uses HSETNX**: These are correctly different operations — `save()` is the Protocol bulk-write path (overwrite semantics, not on hot path), `mark_seen()` is the pipeline write path (set-if-absent semantics, write-once `first_seen_at`). The test `test_write_once_semantics_via_hsetnx` explicitly asserts `hset.assert_not_called()` after `mark_seen`, confirming the distinction is intentional and tested.
  FINAL: Intentional design, not a bug. Only `save()` (Protocol method) uses HSET; `mark_seen()` (pipeline hot path) always uses HSETNX.
  WATCH: Future maintainers must not conflate these two write paths.

- **`except StateError: raise` guard in each method**: Prevents double-wrapping when `StateError` propagates through nested try/except blocks. Cosmetically redundant on the hot path but correct and consistent with the defensive style.
  FINAL: Correct. No action needed.
