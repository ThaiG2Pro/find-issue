**Code review of `_do_post_with_retry`** (lines ~490-513):


### 2. Backoff Formula (AC-001-006 / AC-001-007)

**Code review of backoff calculation** (lines ~507-508):
```python
backoff = self._backoff_base * (2**attempt)
wait = max(retry_after, backoff) if retry_after is not None else backoff
```

- `attempt` starts at `0` (line ~478: `attempt = 0`)
- First retry wait: `backoff_base * 2**0 = backoff_base = 1.0` ‚úÖ
- Sequence with `backoff_base=1.0`: `[1.0, 2.0, 4.0]` ‚Äî pinned by `test_backoff_growth_follows_formula` ‚úÖ
- `max(retry_after, backoff)` is exact AC-001-006 spec formula ‚úÖ
- `retry_after is not None` guard: only use `max()` when `_parse_retry_after` returned a value ‚úÖ

`test_retry_after_floors_backoff`: `Retry-After: 5` with `backoff_base=1.0, attempt=0` ‚Üí `max(5, 1.0) = 5.0` ‚Üí `fake_sleep.assert_called_once_with(5.0)` ‚úÖ

### 5. Subclass Ordering ‚Äî TimeoutException before RequestError

`_do_post_with_retry` catches `httpx.TimeoutException` in the first `except` clause, then `httpx.RequestError` in the second. Since `TimeoutException` is a subclass of `RequestError` in httpx, this ordering is mandatory and correct (consistent with v2-005-push-delivery lesson in memory). ‚úÖ
