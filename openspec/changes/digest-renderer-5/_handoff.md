## 1. Key Decisions (with reasoning)
- ADR-001 — pure `render()` free function + thin `MarkdownDigestRenderer` adapter that delegates to it.
  WHY: keeps the transform mockless-testable as a plain function while preserving a role-named port for S6/S7 wiring; mirrors the summarizer adapter style. `Digest` model untouched (user Q1=A).
  REJECTED: class-only adapter (forces instantiation to test a pure fn); function-only (breaks ports/adapters convention).
- ADR-002 — `DigestRenderer(Protocol)` in `ports.py`, single `render(self, items, *, lookback_days) -> str`, structural impl.
  WHY: matches existing role-named Protocol ports; one new port, no existing Protocol changed (INT-5-001).
  REJECTED: ABC base class; adding `render` to the existing `Delivery` port (conflates S5/S6).
- ADR-003 — determinism via `dict[repo]->dict[group_key]->list`; repos via `sorted(keys, key=str.lower)`; groups via fixed `GROUP_ORDER` + `Khác`; items in append/input order; **NO `set`**.
  WHY: eliminates set/dict-order leakage (RF-1 core risk); byte-for-byte output (AC-5-004); input order preserved within group (AC-5-007).
  REJECTED: sort-whole-list + groupby (easy to accidentally re-sort items); set-then-sort (the exact determinism trap).
- ADR-004 — unknown `item_type` → single trailing `### Khác (N)` group per repo.
  WHY: never drop an item (AC-5-020/BR-5-010); simplest for V1 issues-only.
  REJECTED: per-unknown-type sub-groups (over-engineered); dropping unknown types.
- ADR-005 — NO `openapi.yaml` (CLI-only, no HTTP API). Deliberate ADR-justified deviation from R5/R9.
  WHY: no HTTP surface; INT-5-001 seam is a Python Protocol. Precedent: collector ADR-007, state ADR-004.
  REJECTED: stub openapi.yaml fabricating a non-existent HTTP API.

## 2. Contentious Points

  FINAL: logic lives in the free function; class delegates one line. Both exported from `render/__init__.py`.
  WATCH: do NOT move logic into the class method or duplicate it; do NOT add I/O methods to the port (S6's job).
- ADR-005 (no openapi.yaml): a documented R5 deviation, not an omission.
  FINAL: §API Design documents the Python contract instead. Cross-artifact-audit endpoint check is N/A (0 endpoints = 0 paths).
  WATCH: don't let an audit flag "missing openapi.yaml" as CRITICAL — it's the CLI-only exception.

## 4. Risky Areas
- **Determinism (RF-1, High/functional)** — the whole game. Use the dict-of-dict grouping; sort ONLY repo keys, ONLY by `str.lower`; NEVER use a `set`. QA must assert byte-equality on double-render (EC-006) AND order-independence on shuffled input (EC-007).
- **Empty-field line permutations (RF-2)** — title/url/summary each independently empty (and combined). `_build_item_line` must join present segments so AC-5-012/015/016/017 byte-match exactly. Test each permutation; verify spacing.
- **Import-isolation (AC-5-003)** — needs a STATIC test (`ast`/import inspection of `render/renderer.py` + `__init__.py`), not just runtime. Mirror summarizer AC-4-021. This is flagged for S5 QA explicitly.
- **Line-format byte exactness** — AC-5-012 pins `- #123 "Fix bug" — Handle null pointer. [link](https://gh/123)`. Build and eyeball at the mid-checkpoint (task 5.1) before writing the full test suite.
