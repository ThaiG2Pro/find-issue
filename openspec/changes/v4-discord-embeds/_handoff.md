# Handoff: S3 → S4
Generated by: architect | Date: 2026-07-12

## 1. Key Decisions (with reasoning)
- ADR-001 `_parse_sections`: extract one shared `## `-boundary splitter → `list[{title, body}]`, `[]` on no section.
  WHY: one boundary rule for both plain-text and embed paths prevents drift; `[]` is the single explicit fallback trigger.
  REJECTED: an independent embed-only regex splitter (two rules that drift).
- ADR-002 `_repo_color` via `hashlib.md5(slug) % len(palette)`.
  WHY: builtin `hash()` is PYTHONHASHSEED-salted per process → same repo would get a different color each run → breaks the idempotency non-negotiable. md5 is a stable non-security integer digest.
  REJECTED: builtin `hash()`.
- ADR-003 `_build_embeds`: one embed/section, description ≤4096 **code points**, over-length body line-split (reuse `_split_lines` at 4096); footer `OSS Pulse • {ts}`.
  WHY: reuse the proven two-level splitter; drop no content (vs truncate).
  REJECTED: ellipsis truncation.
- ADR-004 `_batch_embeds`: ≤10-embed batches POSTed sequentially via a payload-parameterized `_post_one`.
  WHY: reuse the single timeout + URL-never-leaked error path; parameterize one helper, don't fork.
  REJECTED: a separate embed-POST method duplicating the try/except.
- ADR-005 `deliver()` switch: `use_embeds and _parse_sections()!=[]` → embeds, else plain text; `false` byte-identical.
  WHY: structural `[]`-trigger fallback means embed formatting alone never fails a run.
  REJECTED: flag-based fallback; raising on an unformattable embed.
- ADR-006 `_validate_discord_use_embeds`: `type(v) is not bool` → `ConfigError`, default `False`.
  WHY: mirrors the existing `_validate_delta` bool-trap; fail-fast at load.
  REJECTED: `isinstance` (accepts `bool`⊂`int`); validating at deliver time.

## 2. Contentious Points
- AC-V4-001-002 (color): the ONLY genuinely-hard correctness point. Design pins `hashlib`, not `hash()`.
  FINAL: ADR-002 + test 2.3 asserts cross-invocation stability.
  WATCH: any use of builtin `hash()` in `_repo_color` is a hard reject.
- ADR-004 touches `_post_one`'s signature (adds a JSON-body param).
  FINAL: parameterize, keep one try/except.
  WATCH: the embed branch must NOT `str(exc)`/`repr(request)` — reuse the status/type-only message composition, or the webhook URL leaks (T1).

## 3. Implicit Assumptions
- The renderer emits the section header exactly as the embed title needs (`## {repo} — {N} ngày qua`), so the title = header line minus `## ` with no date re-computation (confirmed from proposal + renderer read).
- Palette hues are an implementation detail; the AC pins only determinism + palette membership, not specific colors — dev may adjust hues.
- Footer timestamp is delivery-time UTC ISO-8601, intentionally non-idempotent but cosmetic (delivery has no dedupe); content/description determinism is unaffected.

## 4. Risky Areas
- `_repo_color`: verify `hashlib` (not `hash()`) and palette membership — idempotency non-negotiable.
- The 4096 split: verify it measures `len(str)` code points — a body ≤4096 chars but >4096 UTF-8 bytes must NOT be split (test 2.4).
- Parameterized `_post_one`: verify no URL in any `DeliveryError` message and the timeout still bounds every batched request (test 2.6).
- Fallback: verify `_parse_sections("No new items…")` → `[]` → plain-text POST (test 2.5).
